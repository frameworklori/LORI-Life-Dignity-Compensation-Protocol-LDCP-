# LDPC 分層實施框架  
## 為系統穩定而設計的解耦(分散風險)架構

---

## 1. 框架目標

本文件定義一套 **分層且解耦(分散風險)的結構設計**，用於 LDPC 在大規模技術轉型期間管理系統性風險。

本框架並 **不強制要求採用**。  
它規範的是：在何種 **設計約束** 之下，轉型機制能在法律、財務與制度層面保持可存續性。

---

## 2. 分層結構概覽

### 第 0 層 — 普世公共投資（固定）

**性質：**  
無條件、非掠奪性的公共基礎設施投資。

**例子：**  
- 公共教育基礎設施  
- 圖書館與數位存取資源  
- 學費減免機制  
- 開放的轉型資源

**設計目的：**  
- 建立尊嚴底線  
- 防止排除或脅迫  
- 吸收全系統層級的轉型衝擊

---

### 第 1 層 — 條件式利息減免（可變動）

**性質：**  
針對性地緩解財務波動。

**運作機制：**  
- 僅減免利息（不減免本金）  
- 透過機構認證的轉型途徑，以「選擇加入」方式參與

**設計目的：**  
- 降低違約連鎖反應  
- 穩定高風險族群，而不宣稱因果責任歸屬  
- 使揭露風險保持在可控範圍與時間界限內

---

### 第 2 層 — 自願性生態系支援（選配）

**性質：**  
不具約束力的補充資源。

**例子：**  
- 工具、證書、平台點數／額度

**設計目的：**  
- 加速復原，而不形成依賴  
- 絕不能作為進入第 0 層或第 1 層的資格門檻

---

## 3. 核心設計原則

> 尊嚴是無條件的。  
> 參與只影響速度，不影響價值。

---

## 4. 解耦與機構中立性

為確保在法律與政治上的可存續性，LDPC 強制實施嚴格的解耦設計：

- **資金提供者對個人資格審核不享有任何裁量權。**  
- **實際的門檻與審查權限僅保留給公共機構或獨立第三方機構。**  
- 資金來源、資格判定與具體執行在結構上相互分離。

此種分離可防止：  
- 企業介入個案層級的裁決，  
- 因個別化請求而擴張責任範圍，  
- 以及源自個案爭議的聲譽風險暴露。

---

### 第 0 層 — 普及性公共投資與 UBI 穩定底線（固定）

第 0 層由兩個互補的組成部分構成：

1. **公共基礎設施投資**  
   （教育、圖書館、學費減免、數位存取）

2. **UBI 穩定性流動性底線（UBI-Stable Liquidity Floor）**  
   一種低水位、固定、無條件現金流設計，其目的是：
   - 維持最低限度的債務償付能力，  
   - 防止收入突然中斷，  
   - 在轉型期間穩定家庭現金流。

UBI-Stable 在設計上會刻意**低於維生工資水準**，其存在僅作為一種  
**金融穩定工具**，而非用來取代生計的收入來源。

**框架結束**
